01 🔍 Embeddings + البحث الدلالي

يحوّل كل مقال إلى "بصمة رقمية" تفهم المعنى، مش الكلمات. أساس كل ميزات الذكاء في منظومتك.

💡 إيش يسوي ببساطة
التشبيه: موظف قرأ كل مقالاتك وفاهمها
كل مقال يتحوّل لـ vector (مصفوفة أرقام تمثّل المعنى). المقالات اللي معناها قريب، أرقامها قريبة. النظام يقارن الأرقام، يكتشف ترابط ما اكتشفته أنت. الفرق عن البحث العادي: لو كتبت "سيارة"، يجيب لك مقال فيه "عربية" لأن المعنى واحد.
🎯 استخدامات عملية في مدوني
  • ✓
    اقتراح روابط داخلية تلقائياًالكاتب يخلص مقال → النظام يقترح: "ضع رابط لمقال X لأنه مرتبط ٨٧٪". يخدم الـ SEO golden rule.
  • ✓
    كشف التكرار قبل النشرقبل ما يُنشر المقال، النظام يحذّر: "هذا الموضوع كتبته من ٦ شهور بنفس الزاوية، حدّث أو اربط".
  • ✓
    اقتراحات للقارئ في نهاية المقال"قد يعجبك أيضاً..." مبني على المعنى مش الكلمات. يزيد time-on-site بشكل ملحوظ.
  • ✓
    بحث ذكي للزائرالزائر يكتب "أبغى مقاول بنا" → يلقي نتائج "مقاولات + شركات تشييد + بناء فلل" حتى لو الكلمات مختلفة.
  • ✓
    ربط مقال بعميل/صناعة تلقائياًمقال جديد عن "تنظيف الأسنان" → النظام يربطه تلقائياً بالعميل "عيادة بلسم" + صناعة "الرعاية الصحية".
🔧 خطوات الربط مع مدوني
  1. إضافة حقل embedding في Article modelنضيف embedding: Float[] في schema.prisma — MongoDB يدعم arrays مباشرة.
  2. عند نشر/تحديث مقالserver action في dataLayer يستدعي models.embedContent ويحفظ النتيجة في DB.
  3. توليد embeddings للمقالات الحاليةسكريبت scripts/embed-all-articles.mjs يمشي على ٢٠٠ مقال موجود مرة واحدة.
  4. API endpoint للبحث الدلاليendpoint جديد /api/search/semantic يحسب cosine distance بين السؤال والمقالات.
  5. UI: اقتراحات داخل article editortab جديد "روابط مقترحة" في admin يعرض ٥ مقالات مرتبطة مع نسبة التطابق.
  6. UI: "قد يعجبك" في نهاية كل مقال على modonty.comcomponent جديد RelatedArticles يستدعي endpoint البحث الدلالي.
💻 مثال كود مبسّط
// عند حفظ مقال جديد
import { GoogleGenAI } from "@google/genai";

const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY });

const embedding = await ai.models.embedContent({
  model: "gemini-embedding-001",
  content: article.title + "\n" + article.body,
});

await db.article.update({
  where: { id: article.id },
  data: { embedding: embedding.values },
});

الخيار A — بسيط موصى به للبداية

نجيب كل المقالات، نحسب التشابه في الـ code التطبيقي.

المميزات:
  • يشتغل على أي MongoDB (حتى free tier)
  • لا يحتاج تغيير في إعدادات Atlas
  • سهل التطبيق
العيوب:
  • مع +١٠٠٠ مقال، يصير بطيء
  • يستهلك ذاكرة في الـ server
🎯 المناسب: مدوني الآن (٢٠٠ مقال) — ممتاز

الخيار B — متقدّم (لما تكبر)

MongoDB Atlas Vector Search — feature مدمج يبحث في embeddings بسرعة الضوء.

المميزات:
  • سريع جداً (مللي ثانية حتى مع ١٠٠ ألف مقال)
  • MongoDB يسوي العمليات بدل الـ code
العيوب:
  • يحتاج إنشاء Vector Index في Atlas dashboard
  • يحتاج تعديل في الـ queries
🎯 المناسب: لما توصل +٥٠٠٠ مقال

02 📚 Long Context — الذاكرة الطويلة

Gemini 2.0 يقدر يستوعب مليون token دفعة واحدة. يعني تعطيه ١٠٠ مقال + كل قواعدك دفعة وحدة ويجاوبك على أساس المجموع كله.

💡 إيش يسوي ببساطة
التشبيه: موظف يقرأ ١٠٠ مقال في دقيقة ويعطيك تقرير
عادةً نسأل Gemini عن مقال واحد. مع Long Context تعطيه مكتبة كاملة + سياسات الكتابة + brand guidelines، ويفهم كل دا كـ context. تقدر تسأله أسئلة استراتيجية تخص مجموع المحتوى.
🎯 استخدامات عملية في مدوني
  • ✓
    Brand Voice Audit شهري"اقرأ كل مقالات الشهر، هل النبرة متناسقة؟ أعطني تقرير + ٥ مقالات تخالف الـ tone".
  • ✓
    كشف التناقضاتلو ذكرت "+١٢٠ عميل" في مقال + "+٨٠ عميل" في مقال آخر، Gemini يكشف التناقض تلقائياً.
  • ✓
    Content Gap Analysis"بناءً على ٢٠٠ مقال، ما هي ٥ مواضيع مفقودة من منظومة مدوني؟ مع ترتيب حسب فرص SEO".
  • ✓
    Pre-publish style checkقبل نشر مقال جديد، Gemini يقارنه بـ ٢٠ مقال سابق ويقول: "هذي الفقرة مش بنبرة مدوني، اقترح بديل".
  • ✓
    تحليل تطور المحتوى"كيف تطوّر أسلوبي من ٢٠٢٥ إلى ٢٠٢٦؟ هل صار أوضح أم أعقد؟"
🔧 خطوات الربط مع مدوني
  1. إنشاء UI tab في adminصفحة /admin/brand-audit بثلاثة أزرار: Run Audit · Style Check · Gap Analysis.
  2. Server action يجمع المحتوىيأخذ آخر ٣٠ مقال + ملف brand-guidelines.md + قائمة المواضيع المنشورة.
  3. استدعاء Gemini مع كل المحتوىAPI call واحد بـ gemini-2.0-flash مع context كامل + prompt محدد.
  4. حفظ التقرير كـ snapshotmodel جديد BrandAudit في Prisma يحفظ التقارير الشهرية.
  5. UI لعرض التقريرصفحة /admin/brand-audit/[id] تعرض التقرير بشكل منظم.
  6. Cron job اختيارييشغّل audit شهري تلقائياً ويرسله للأدمن بالإيميل.
💻 مثال كود مبسّط
// تجميع آخر 30 مقال
const articles = await db.article.findMany({
  where: { status: "PUBLISHED" },
  take: 30,
  orderBy: { publishedAt: "desc" },

const context = articles.map(a =>
  `Title: ${a.title}\nBody: ${a.body}`
).join("\n---\n");

const result = await ai.models.generateContent({
  model: "gemini-2.0-flash",
  contents: [
    { role: "user", parts: [{ text: context }] },
    { role: "user", parts: [{ text: "حلّل اتساق النبرة..." }] },
  ],
});

03 🖼️ Image Understanding — يفهم الصور

Gemini يشوف الصورة ويوصفها بدقة عربية: اللون، التكوين، حتى النص الموجود فيها. أداة قوية للـ accessibility والـ SEO.

💡 إيش يسوي ببساطة
التشبيه: كأن لـ Gemini عينين تشوف الصورة وتفهمها
تعطيه صورة، يرجع لك وصف نصّي بالعربي. يحلّل الـ context (هل هي طبية؟ تجارية؟ فاخرة؟)، الجودة (مضاءة؟ مظلمة؟ blurry؟)، ويستخرج أي نص داخل الصورة.
🎯 استخدامات عملية في مدوني
  • ✓
    Alt Text تلقائي للصوركل صورة تُرفع → Gemini يكتب alt عربي دقيق → Google يفهم → ترتيب في بحث الصور. هذا requirement لـ WCAG accessibility.
  • ✓
    فحص جودة الصورة قبل النشر"الصورة blurry، استبدلها" أو "الإضاءة ضعيفة" — منع نشر صور رديئة الجودة.
  • ✓
    استخراج نص من screenshotsالـ admin يلصق لقطة شاشة فيها كلام → Gemini يحوّلها text → جاهز للنسخ.
  • ✓
    كشف logos العملاء تلقائياًصورة فيها logo عميل → النظام يتعرّف عليه ويربط الصورة بالعميل.
  • ✓
    اقتراح cropping"اقص هنا للحصول على hero image مثالي" — يحدد الـ focal point.
🔧 خطوات الربط مع مدوني
  1. تعديل upload flow في adminبعد upload للـ Cloudinary، نستدعي Gemini مع URL الصورة.
  2. Server action: generateAltText(mediaId)تأخذ ID الصورة، تجلب URL من DB، ترسل لـ Gemini مع prompt محدد.
  3. Prompt مصاغ بدقة"اكتب alt text عربي مختصر (٨٠ حرف أقصى) يصف الصورة + المعنى المقصود. لا تستخدم 'صورة لـ' في البداية."

04 🔧 JSON-LD Auto-Fix

JSON-LD هو "بطاقة تعريف" المقال اللي Google يقرأها قبل المحتوى. حالياً لو فيها warning، الـ admin يحلها يدوياً. Gemini يحلها تلقائياً.

💡 إيش يسوي ببساطة
التشبيه: مهندس جودة يفحص الـ schema ويصلحها بدلك
Gemini يفهم schema.org كاملاً + قواعد Google Rich Results. يقرأ المقال + الـ schema الحالي + قائمة warnings → يولّد schema مُصلَح ومُحسَّن. يخدم مباشرة قاعدة مدوني الذهبية: "JSON-LD = code responsibility, never admin task".
🎯 استخدامات عملية في مدوني

    Auto-Fix فوري عند Warningadmin يفتح المقال → ⚠️ warning → يضغط "Auto-Fix with AI" → Gemini يصلحها + يعرض diff.
  • ✓

  • ✓
    Validation تلقائي قبل النشرفي الـ publish pipeline، Gemini يفحص + يصلح + يوافق قبل أن يصل إلى Google.
  • ✓
    Bulk audit للمقالات القديمة"افحص ٢٠٠ مقال، أعطني قائمة كل المقالات اللي schema تبعها فيه مشاكل + اقتراحات للإصلاح".
  • ✓
    Schema enhancement بناءً على ميزات Google الجديدةلما Google يضيف schema جديد (مثل LearningResource)، Gemini يعرف ويقترح الإضافة.

14 💾 Context Caching

يحفظ الـ context المتكرر (مثل brand guidelines) في cache → التكلفة تنزل ٧٥٪ على الـ calls المتكررة.

💡 إيش يسوي ببساطة
التشبيه: بدل ما تشرح القاعدة لكل موظف جديد، تسجّلها مرة وحدة
Gemini يحفظ الـ context الطويل (brand guidelines + style guide + schema knowledge) في cache. كل call تالي يستخدم الـ cache → ما يحتاج يدفع على الـ tokens المكررة. التوفير ٧٥٪.
💼 سيناريو من مدوني
❌ المشكلة الفعلية

كل brand audit شهري (#2 في القدرات) يرسل لـ Gemini:

  • brand guidelines (~5K tokens)
  • past audits (~10K tokens)
  • المقالات الجديدة (~30K tokens)

الـ guidelines والـ past audits ما يتغيّرون — لكن ندفع عليهم في كل call شهرياً.

💡 الحل

نخزّن الـ guidelines + past audits في cache (Gemini يحفظهم لنا). كل audit جديد:

  • نشير للـ cache_id (مجاناً تقريباً)
  • نرسل بس المحتوى الجديد
  • توفير ٧٥٪ على الـ tokens المتكررة
📊 الفائدة بالأرقام
  • brand audit: من $0.05 → $0.012
  • Function Calling Agent: من $0.10/مقال → $0.03/مقال
  • JSON-LD auto-fix: نفس التوفير
  • يخدم كل الميزات الكبيرة اللي تستخدم context طويل

15 🛡️ Content Moderation
