الحاجة

أكثر من موظفة تعمل على الأدمن، وأكثر من كاتب محتوى — ومحتاج تعرف: مين عمل إيش؟

اليوم لو انحذف مقال أو اتنشر شي بالغلط، ما في طريقة تعرف مين عمله. كل العمليات مجهولة الفاعل.

🔴 لا — كلمة المرور لا تُسجَّل إطلاقاً

طلبك ذكر «معرّف المستخدم + الاسم + كلمة المرور». الأولان تمام، والثالث ممنوع منعاً باتاً — وهذه ليست وجهة نظر:

  • كلمات المرور مشفّرة بـ bcrypt (تشفير لا رجعة فيه) — نحن أنفسنا لا نقدر نقرأها.
  • تسجيلها = أي شخص يفتح السجل يسرق كل الحسابات.
  • مخالفة صريحة لنظام حماية البيانات الشخصية السعودي (PDPL).
  • ولا تضيف شيئاً لهدفك: «مين فعل هذا؟» يجاوبه المعرّف والإيميل بالكامل.

البديل الذي اقترحتَه أنت وهو الصحيح: userId + email.

① الجدول

// AuditLog — من فعل ماذا، ومتى userId String ← المعرّف (المفتاح الحقيقي) userEmail String ← لقطة نصية وقت الفعل action String ← "article.publish" · "client.delete" entity String ← Article · Client · Category … entityId String? ← على مَن وقع الفعل summary String? ← "نشر: أفضل واكس شعر" createdAt DateTime

لماذا الإيميل نصاً وليس علاقة فقط؟ لأنه يجاوب «مين؟» فوراً بلا استعلام إضافي — ويبقى محفوظاً حتى لو تركت الموظفة العمل وحُذف حسابها. السجل يجب أن يصمد بعد صاحبه.

② الدالة — سطر واحد بعد كل عملية

// بعد نجاح العملية مباشرة await logAction("article.delete", { entityId: id, summary: article.title });

الدالة تقرأ الهوية من الجلسة بنفسها — لا نمرّر لها شيئاً عن المستخدم. وفشل التسجيل لا يُفشل العملية أبداً (تُبتلع أخطاؤه بصمت).
