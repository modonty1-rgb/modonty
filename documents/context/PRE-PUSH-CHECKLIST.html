🔑 القاعدة اللي تمنع القفل

الدخول صار يقرأ من db.staff، وstaff على الإنتاج فاضي. لا تبدّل الدخول قبل ما تتعبّى staff. فالترتيب إجباري: طبّق schema → اجعل الدخول يقرأ staff مع fallback لـ user (بلا قفل أثناء الانتقال) → عبّي staff بالترحيل → تحقق → دوّر المفتاح → شِل الـ fallback.

Phase 1 — جهّز الطاقم (كود ناقص لازم يُبنى)

Phase 2 — بوابات الجودة

Phase 3 — نسخة احتياطية + إصدار + دفع

Phase 4 — تطبيق الطاقم على الإنتاج (بالترتيب)

Phase 5 — بعد النشر (تحقق شامل)
