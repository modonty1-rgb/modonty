## A1 — صفحة /about: عنوان `<title>` = Lorem Ipsum

**الصفحة**: `/about`
**المشكلة**: عنوان الصفحة في `<title>` هو:
```
Magnam incidunt irure delectus pariatur Magna ullamco et - مودونتي | مودونتي
```
هذا نص seed data لاتيني يظهر في Google ومشاركات السوشال والمتصفح.

**الدليل**: عند fetch الصفحة، العنوان الراجع من السيرفر هو هذا النص.

**الملف المتوقع**: الـ `<title>` يجي من الـ DB عبر admin panel (حسب قواعد الـ audit — SEO content from DB).

**الإصلاح**: تحديث عنوان صفحة "من نحن" في admin panel إلى: `من نحن — مودونتي | منصة المدونات الاحترافية`

**الخطورة**: 🔴 حرج (SEO + مصداقية)

**الصفحة**: `/clients`
**المشكلة**: 3 عملاء تجريبيين ظاهرين بأسمائهم الكاملة:

**الإصلاح**: من admin panel — إما حذف العملاء التجريبيين أو إخفاؤهم (flag: isTest/isVisible).

**الخطورة**: 🔴 حرج (مصداقية)

## A3 — محتوى الصفحة الرئيسية يُحمّل Client-Side

**الصفحة**: `/` و `/trending` و `/categories`
**المشكلة**: عند طلب الصفحة من السيرفر، المحتوى الراجع هو:
```
## جاري تحميل آخر المقالات
```
هذا يعني أن المقالات تُحمّل بـ JavaScript بعد وصول الصفحة (Client-Side Rendering).

**التأثير**:
- Google قد لا يفهرس المحتوى الرئيسي (المقالات) لأنه غير موجود في الـ initial HTML
- المستخدم يرى "جاري تحميل" قبل ظهور المحتوى
- يظهر على كل الصفحات: الرئيسية + الرائجة + الفئات

**الإصلاح**: تحويل الـ data fetching لـ Server-Side:
- استخدم server component مع `await` لجلب المقالات
- أو استخدم `loading.tsx` مع skeleton loaders بدل نص "جاري تحميل"

**ملاحظة**: صفحة `/about` و `/clients` تُرجع محتواها بشكل صحيح من السيرفر — المشكلة فقط في الصفحات التي تعرض قوائم مقالات.

**الخطورة**: 🔴 حرج (SEO)

## A5 — التواريخ لا تظهر — تعرض "..."

**الصفحة**: `/` و `/articles/*`
**المشكلة**: في بطاقات المقالات، بدل تاريخ النشر يظهر:
```
...•⏱️ 2 دقيقة
```
الـ `...` يجب أن يكون تاريخ (مثل: "2 أبريل 2026"). في صفحة المقال أيضاً:
```
آخر تحديث: ...
```

**السبب المحتمل**: التاريخ يُحسب client-side وعند الـ SSR يظهر كـ `...` placeholder.

**الإصلاح**: استخدم `suppressHydrationWarning` على عنصر التاريخ، أو اعرض التاريخ بصيغة ISO من السيرفر وحوّله client-side.

**الخطورة**: 🟡 متوسط (UX + SEO)

## A6 — صور بدون تحديد عرض في Cloudinary

**الصفحة**: `/` و `/articles/*`
**المشكلة**: بعض صور Cloudinary فيها width transform وبعضها لا:

| الصورة | Cloudinary Transform | ملاحظة |
|--------|---------------------|--------|
| Logo | `w_300,c_limit` | ✅ محدد |
| Avatar | `w_96,c_limit` | ✅ محدد |
| صورة عميل (future-logo) | `f_auto,q_auto` فقط — **بدون width** | ❌ ممكن تكون كبيرة |
| صورة مقال hero (future-og) | `f_auto,q_auto` فقط — **بدون width** | ❌ ممكن تكون كبيرة |

**تصحيح عن التقرير السابق**: الـ avatar صوره محمية بـ `w_96,c_limit` من Cloudinary — **مو مشكلة bandwidth**. المشكلة الفعلية في صور المقالات والعملاء اللي ما عندها width transform.

**الإصلاح**: أضف width transform لكل صور Cloudinary:
```
// صور العملاء (logo)
/upload/f_auto,q_auto,w_200,c_limit/

// صور المقالات (hero/og)
/upload/f_auto,q_auto,w_800,c_limit/

// صور المقالات في البطاقات
/upload/f_auto,q_auto,w_400,c_limit/
```

**ملاحظة**: مسار الصور يحتوي على `/seed-data/` — مما يؤكد أنها بيانات تجريبية.

**الخطورة**: 🟡 متوسط (Performance)

## B4 — Sidebar الرئيسية مزدحم

**الصفحة**: `/`
**المشكلة**: الـ sidebar يحتوي على 5 أقسام:
1. تحليلات (المقالات 2 + التفاعلات ٤ + ... )
2. الفئات (14 فئة — معظمها 0 مقالات)
3. جديد مودونتي (مقالتين — نفس المقالات في المحتوى الرئيسي)
4. شركاء النجاح (نفس المقالتين مرة ثالثة!)
5. المزيد (أخبار + من نحن + اشترك + ملاحظات)

**المشكلة**:
- نفس المقالتين تتكرر 3 مرات: في المحتوى الرئيسي + جديد مودونتي + شركاء النجاح
- 14 فئة بـ 0 مقالات — تعطي انطباع أن المنصة فارغة
- قسم التحليلات يعرض أرقام ضعيفة (٤ تفاعلات، ٥ مشاهدات)

**الإصلاح**: 
- إخفاء الفئات اللي فيها 0 مقالات
- إزالة التكرار (جديد مودونتي + شركاء النجاح يعرضون نفس المحتوى)
- إخفاء التحليلات لما الأرقام قليلة (أو إخفاؤها عن الزوار غير المسجلين)

**الخطورة**: 🟡 متوسط (UX)

## ✅ ما هو جيد (مؤكد)

## ❌ الفجوات المؤكدة

### C1 — صور Avatar بدون alt وصفي
**الدليل**: الـ avatar في الـ navbar:
```html
<img alt="" src="...modontyAvatar...">
```
`alt=""` يعني decorative — لكن هذي صورة حساب المستخدم، يجب أن يكون لها `alt` وصفي.

### C2 — أرقام التفاعل بدون سياق
**الدليل**: في بطاقة المقال يظهر `10035` كرابط بدون أي `aria-label` يوضح إنه عدد المشاهدات. وكذلك `0`, `0`, `0`, `0` بدون تسميات.

### C3 — "نسخة صوتية" بدون aria-label
**الدليل**: النص "نسخة صوتية" يظهر في كل بطاقة مقال — لو كان زر يحتاج `aria-label="تشغيل النسخة الصوتية للمقال"`.

### C4 — "اسأل العميل" في صفحة المقال
**الدليل**: زر موجود في صفحة المقال — يحتاج `aria-label` يوضح السياق.

### C5 — حقل البحث = رابط مو input
**الدليل**: "ابحث..." هو `<a href="/search">` — المستخدم يتوقع حقل بحث فوري. من حيث الـ accessibility ما فيه مشكلة، لكن من حيث الـ UX المتوقع يحتاج تقييم.

### C6 — تحتاج فحص يدوي (لا أقدر أتحقق من web fetch)
- Keyboard navigation الكامل
- Focus states على كل العناصر التفاعلية
- Color contrast ratios
- Slider في /clients — Arrow keys support
- Screen reader compatibility

# القسم D: قواعد الـ UI — Typography & Content
# ════════════════════════════════════════════════════

### D1 — "..." بدل "…" (ellipsis)
**الدليل**: في بطاقات المقالات: `...•⏱️ 2 دقيقة`
**القاعدة**: استخدم `…` (HTML entity `&hellip;`) بدل ثلاث نقاط.

### D2 — لا `font-variant-numeric: tabular-nums` على الأرقام
**الدليل**: صفحة `/clients` تعرض أرقام (مقالات: 4, مشاهدات: 141, مشتركون: 17) — بدون tabular-nums الأرقام لا تنتظم عمودياً.

### D3 — "اقرأ المزيد" — generic CTA
**الدليل**: كل بطاقة مقال فيها "اقرأ المزيد" — نص غير محدد.
**الأفضل**: استخدم عنوان المقال كـ link بدل زر منفصل، أو "اقرأ المقال".

### D4 — وصف الشركات بقالب موحد
**الدليل**: كل شركات الـ seed data تستخدم نفس القالب:
```
"[الاسم] هي [النوع] متخصصة في [المدينة]. نقدم حلول احترافية وعالية الجودة لعملائنا في المنطقة."
```
هذا مقبول لـ seed data لكن عند الإنتاج الحقيقي يجب أن يكون كل وصف فريد.

### D5 — تناقض لغوي
**الدليل**:
- وصف الكاتب بالإنجليزي: "Modonty is a leading content platform..."
- بقية الموقع بالعربي
- فئات في `/categories` بالإنجليزي: "On-Page SEO", "Technical SEO", "E-E-A-T"
- هذا مقبول لمصطلحات تقنية لكن الأوصاف يجب أن تكون بالعربي
