الحالي: الضغط يرمي الزائر مباشرة لـ /users/login (دخول، مش تسجيل)

سلوك الضغط — أيّهما؟

أ — توجيه مباشر لصفحة التسجيل فورًا (بلا شرح). أسرع، لكن مفاجئ وبلا سياق.
ب — prompt في اللحظة الموصى يشرح الفائدة («احفظ مقالاتك») ثم زر «اشترك مجاناً». يحوّل أعلى لأنه يربط الفعل بالقيمة (NN/g: لا ترمِ المستخدم في مسار بلا سياق).
مواصفة المطابقة (HTML ↔ كود حقيقي):
• الـ prompt = مكوّن shadcn Dialog (أو Drawer على الجوال) · الحواف rounded-2xl من توكِن النظام.
• زر الأساسي = variant="default" (أزرق العلامة #3030FF) → /users/register?callbackUrl=<الصفحة>.
• رابط «دخول» = نص خافت → /users/login?callbackUrl=....
• العنوان والأيقونة سياقيان حسب الفعل (حفظ/متابعة/إعجاب/تعليق/تقييم).
• أزرار المقال تصير مفعّلة (تشيل disabled) — الضغط يفتح الـ prompt بدل لا شيء.
• ألوان الـ mockup الداكنة تمثيلية — التطبيق يستخدم توكِنز الثيم الفعلية من globals.css.
