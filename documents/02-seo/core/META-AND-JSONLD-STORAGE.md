# Meta Tags & JSON-LD Storage in Database

How meta tags and JSON-LD are stored per table (Prisma schema + usage).

---

## Client/Organization

| Storage | Field(s) | Type | Purpose |
|---------|----------|------|---------|
| Individual meta columns | seoTitle, seoDescription, metaRobots, twitterCard, twitterTitle, twitterDescription, twitterSite, canonicalUrl | String? | Editable source of truth |
| Cached meta object | metaTags | Json? | Full meta object generated from columns + media; saved on "Generate SEO" |
| JSON-LD | jsonLdStructuredData, jsonLdLastGenerated, jsonLdValidationReport | String?, DateTime?, Json? | Cached @graph JSON-LD string with validation |

Generated by: `generate-client-seo.ts` → builds metaTags and JSON-LD, updates Client.
Consumed by: Admin client page `generateMetadata()` reads metaTags for Next.js Metadata.

---

## Article

| Storage | Field(s) | Type | Purpose |
|---------|----------|------|---------|
| Individual meta columns | seoTitle, seoDescription, metaRobots, ogType, ogArticleAuthor, ogArticlePublishedTime, ogArticleModifiedTime, twitterCard, twitterSite, twitterCreator, canonicalUrl, alternateLanguages, sitemapPriority, sitemapChangeFreq, breadcrumbPath | String?, DateTime?, Json? | Editable source of truth |
| No metaTags | — | — | Article does NOT store cached meta object |
| Next.js metadata cache | nextjsMetadata, nextjsMetadataLastGenerated | Json?, DateTime? | Cached Next.js Metadata object |
| JSON-LD | jsonLdStructuredData, jsonLdLastGenerated, jsonLdValidationReport, jsonLdVersion, jsonLdHistory, jsonLdDiffSummary | String?, DateTime?, Json?, Int, Json?, String? | Cached @graph with versioning |

Generated by: Article SEO/JSON-LD generation writes jsonLdStructuredData.
Consumed by: Public app builds metadata from individual columns + related Client/Author at request time, or from nextjsMetadata if cached.

## Modonty (App Pages: About, Terms, etc.)

| Storage | Field(s) | Type | Purpose |
|---------|----------|------|---------|
| Individual meta columns | seoTitle, seoDescription, metaRobots, ogTitle, ogDescription, ogType, ogUrl, ogSiteName, ogLocale, ogImage, twitterCard, twitterTitle, twitterDescription, twitterSite, twitterCreator, canonicalUrl, alternateLanguages, sitemapPriority, sitemapChangeFreq, inLanguage | String?, Float?, Json? | Editable source of truth |
| Cached meta object | metaTags | Json? | Generated meta object from page columns + Settings; saved on "Generate SEO" |
| JSON-LD | jsonLdStructuredData, jsonLdLastGenerated, jsonLdValidationReport | String?, DateTime?, Json? | WebPage @graph JSON-LD string |

Generated by: `generate-modonty-page-seo.ts` builds metaTags and JSON-LD.
Consumed by: Modonty app uses stored metaTags and jsonLdStructuredData for page.

---

## Settings (Global Defaults + Optional Site-Level JSON-LD Cache)

- **Defaults:** Settings holds defaults (defaultMetaRobots, defaultOgType, siteName, modontySeoTitle, modontySeoDescription, etc.) used when building meta/JSON-LD.
- **Cached site-level JSON-LD:** Settings has jsonLdStructuredData (String?), jsonLdLastGenerated (DateTime?), jsonLdValidationReport (Json?); used for site-wide/default @graph.
- **No metaTags:** Settings does not store metaTags object.

---

## Pattern Summary

| Table | Individual Meta Columns | Cached Meta Object | Cached JSON-LD |
|-------|--------------------------|--------------------|--------------------|
| Client | ✅ (seo*, twitter*, etc.) | ✅ metaTags (Json) | ✅ jsonLdStructuredData |
| Article | ✅ (seo*, og*, twitter*, etc.) | ❌ | ✅ jsonLdStructuredData; optional nextjsMetadata |
| Modonty | ✅ (seo*, og*, twitter*, etc.) | ✅ metaTags (Json) | ✅ jsonLdStructuredData |
| Settings | Only default/config fields | ❌ | ✅ jsonLdStructuredData |

**Meta tags:** Stored as individual columns (source of truth). Client and Modonty also store generated metaTags Json object for fast metadata output.

**JSON-LD:** Stored as cached string (jsonLdStructuredData) with optional jsonLdValidationReport and versioning (Article only).
