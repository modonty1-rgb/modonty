# مدونتي — الشرح الكامل للنظام
## MODONTY — Complete System Guide

> هذا الملف يشرح مدونتي من الصفر — من هم المستخدمون، كيف يتفاعلون، كيف تتدفق البيانات.

---

## 1. ما هي مدونتي؟

**مدونتي = خدمة تسويق بالمحتوى (Content Marketing as a Service)**

الشركة التي ليس لديها وقت أو فريق لكتابة المحتوى تشترك في مدونتي.
مدونتي تكتب لها مقالات احترافية، تنشرها على موقع عام، وتعطيها لوحة تحكم لمتابعة النتائج.

**بمعنى آخر:**
- الشركة تدفع → مدونتي تكتب → الجمهور يقرأ → الشركة تشوف النتائج

## 5. MODONTY — الموقع العام

### من يدخل عليه؟

```
modonty.com/
│
```

## 10. نظام Lead Scoring — كيف تعرف الشركة من هو العميل المحتمل؟

```
القارئ يتفاعل مع محتوى الشركة:

  قرأ مقالة         → نقاط
  قضى وقت طويل      → نقاط أكثر
  قرأ أكثر من مقالة → نقاط أكثر
  ترك تعليق         → نقاط
  اشترك في النشرة   → نقاط أكثر
  تواصل معها        → أعلى نقطة

              ▼

  HOT  🔴 (70-100 نقطة) ← عميل محتمل جداً — تواصل معه الآن
  WARM 🟡 (40-69 نقطة)  ← مهتم — استمر في إعطائه محتوى
  COLD 🔵 (0-39 نقطة)   ← في مرحلة البحث — اتركه يتصفح

الشركة تشوف هذا في CONSOLE → Leads
```

## 12. الـ SEO System — كيف تظهر المقالات في Google؟

```
Admin يضغط "نشر" →
  ① يُبنى JSON-LD (Schema.org) تلقائياً
  ② يُبنى Next.js Metadata (title, description, OG image)
  ③ يُحفظ في DB كـ cache
  ④ modonty يقرأ الـ cache ويعرضه — بدون عمليات حسابية
  ⑤ Google تقرأ JSON-LD → تفهم المحتوى بشكل أفضل
  ⑥ نتيجة: ظهور أفضل في نتائج البحث
```

## 13. المصطلحات المهمة

| المصطلح | المعنى |
|---------|--------|
| Client | الشركة المشتركة في خدمة مدونتي |
| Article | مقالة كتبها فريق مدونتي لصالح شركة |
| Author | الكاتب من فريق مدونتي |
| Subscriber | شخص اشترك في النشرة البريدية لشركة |
| Lead | قارئ يُشير سلوكه إلى اهتمام تجاري |
| Analytics | بيانات التفاعل والمشاهدات |
| JSON-LD | بيانات منظمة تقرأها Google |
| revalidate | تحديث الـ cache بعد أي تغيير |
| clientId | المعرّف الفريد لكل شركة (يُستخدم لفلترة البيانات) |
| PENDING | في انتظار الموافقة |
| APPROVED | موافق عليه — يظهر للجمهور |
| REJECTED | مرفوض — لا يظهر |

## 17. ملخص سريع — للمصمم أو المطور الجديد

```
إذا كنت جديداً على المشروع، هذا كل ما تحتاج معرفته:

1. ثلاثة تطبيقات — قاعدة بيانات واحدة (MongoDB)
   admin/ → فريق مدونتي
   modonty/ → القراء
   console/ → الشركة المشتركة

2. كل شركة (client) لها clientId فريد
   console يفلتر كل شيء بـ clientId

3. تدفق المقالة:
   Admin ينشئ → Console يوافق → modonty ينشر → Analytics

4. تدفق التعليق:
   modonty يستقبل → DB يحفظ PENDING → Console يوافق → modonty يعرض

5. SEO:
   Admin يولّد JSON-LD + metadata → يُحفظ في DB → modonty يقرأه

6. القاعدة الذهبية:
   أي تعديل في أي تطبيق → تحقق من الثلاثة قبل الكود
```
